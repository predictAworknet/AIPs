# AWP Improvement Proposals (AIPs)

AIPs are formal proposals for changes and improvements to the AWP protocol, subnets, and governance processes.

Inspired by [Ethereum's EIP process](https://eips.ethereum.org/).

## Submit a Proposal

1. Fork this repository
2. Copy `aip-template.md` to `AIPS/aip-XXX.md` (replace XXX with the next number)
3. Fill in the template with your proposal
4. Open a Pull Request

## AIP Types

- **Subnet** — proposals for new subnets or changes to existing subnet behavior
- **Protocol** — changes to the core AWP protocol (emission, staking, governance)
- **Meta** — process changes, governance rules, or informational documents

## AIP Statuses

| Status | Meaning |
|--------|---------|
| Draft | Initial proposal, open for discussion |
| Review | Under formal community review |
| Accepted | Approved by governance, pending implementation |
| Final | Implemented and active |
| Withdrawn | Removed by the author |
